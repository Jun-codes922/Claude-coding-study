#### 바이브 코딩 2주차 정리



##### 효과적인 프롬프트로 AI 활용하기



| 역량 | 설명 |

| --- | --- |

| 바이브 코딩 | AI코딩 파트너를 활용, 자연어 프롬프트로 빠르게 개발 |

| 컴퓨팅 사고 | 문제 분해 →패턴 인식 → 추상화 → 알고리즘적 해결 |

##### 지난 주 복습



\- 바이브 코딩 : 자연어로 AI와 대화하며 코드를 개발하는 방식

\- 클로드 Arifacts : 코드 실행 결과를 실시간 미리 보기로 확인하는 작업 공간

\- 반복 개선)Iteration): 프롬프트 \_> 생성 →개선 사이클



##### &#x20;컴퓨팅 사고



컴퓨팅 사고의 4가지 사고력이 어떻게 적용 되는지 살펴보기



| 사고력 | 의미 | 이번주 적용 |

| --- | --- | --- |

| 분해 | 큰 문제를 작은 단위로 나누기 | 5W1H로 프로젝트를 6가지 질문으로 분해 |

| 패턴 인식 | 반복되는 구조 찾기 | 요청→생성→확인→수정 사이클의 반복 |

| 추상화 | 핵심만 뽑아 정리하기 | 요구사항을 PRD 템플릿으로 추상화 |

| 알고리즘 | 순서대로 절차밟기 | 뼈대→기능→디자인→점검 4단계 |



“4단계 전략”은 곧 컴퓨팅 사고의 실전 적용이다!



##### &#x20;3대역략, 적용



| 역량 | 적용 |

| --- | --- |

| 바이브 코딩 | 클로드에게 프롬프트 포트폴리오 웹페이지 작성 |

바이브 코딩 = 어떻게 만드는가 (도구)



컴퓨팅 사고 = 어떻게 생각하는가 (사고)



프로젝트 관리 =  어떻게 진행하는가(계획)



##### &#x20;프롬프트의 중요성



AI는 방대한 언어 데이터를 학습한 거대한 지식 저장소 이다.



이 거대한 지식 창고와 소통할 수 있는 통로는 프롬프트 창 하나뿐이다.



프롬프트는 내비게이션에 목적지를 입력하는 것과 비슷하다. 만약 “좋은 데 데려다줘”라는 명령에는 정확한 안내를 받을 수 없지만, 구체적으로 “서울역애서 강남역까지 가장 빠른 길”이라고 구체적으로 입력하면 효율적인 루트를 통해 목적지에 도달한다.



제한된 언어로 AI의 잠재력을 최대한 끌어내기 위해서는 명확하고 구체적인프롬트 작성이 필수적임

##### &#x20;좋은프롬프트의 3가지 조건



좋은 프롬프트는 단순히 길거나 화려한 설명이 아니라, \*\*3가지 요소\*\*를 충족해야함 



| 조건 | 설명 | 예시 |

| --- | --- | --- |

| 명확성 | 모호한 표현을 피하고, 원하는 결과를 구체적으로 전달 | 멋진(X)→밝고 깔끔한 스타일 |

| 구체정 | 결과물에 필요한 기능이나 조건을 세부적으로 지시 | 시계와 검색바도 추가해줘 |

| 맥락 제공 | 왜 필요핝;, 어떤 상황에서 활용되는지 알려주기 | 취업용 마케팅 포토폴리오야 |



##### &#x20;5W1H - 체계적인 프롬프트의 핵심 틀



좋은 프롬프트(- 좋은 기획서)를 만들기 위한  체계적인 사고틀이다

| 질문 | 의미 | 역할 |

| --- | --- | --- |

| Why | 왜 만드는 가? | 목적을 명확히 |

| Who | 누구를 위한 것인가? | 대상을 정의 |

| What | 무엇을 만들 것인가? | 기능을 구체화 |

| When | 언제 사용되는가? | 사용 맥락 파악 |

| Where | 어디서 사용되는가? | 사용 환경 정의 |

| How | 어떻게 만들 것인가? | 구현 방법 결정 |



##### &#x20;PRD - 제품 기획서란?



개발자들이 코딩 앞서 작성하는 제품 **기획서(PRD: Product Requirements Document)**이다.



| PRD |  | 좋은 프롬프트 |

| --- | --- | --- |

| 무엇을 만들지 명확히 정의 | = | 해야 할 일을 구체적인 지시 |

| 프로젝트 목적과 배경 지식 | = | 왜 필요한지, 어떤 상황인지 설명 |

| 요구사항 구현 방법 구체화 | = | 필요한 기능과 조건 세부 명시 |



💡 좋은 프롬프트는  잘 만들어진 기획서! 5W1H 템플릿을 활용하면 누구나 체계적인 기획서를 작성할 수 있다



##### &#x20;4단계 포트폴리오 전략



큰 프로젝트를 4단계로 분해 하여 단계별로 AI에게 요청한다.





| 단계 | 목표 | 핵심 활동 |

| --- | --- | --- |

|   1. 뼈대 | HTML 기본 구조 | 섹션 구성, 고유 이름(ID) 부여 |

| 2. 기능 | 실제 데이터 | 경력, 성과, 케이스 스터디 |

| 3. 디자인 | 시각적 완성도 | 레퍼런스, 색상 레이아웃 |

| 4. 점검 | 품질 검증 | PRD   대조, 링크 확인 |



##### &#x20;왜 4단계로 나누는 가?



##### &#x20;한 번에 모든 것을 요청하면?



\- HTML, CSS, JavaScript가 뒤섞인 복잡한 코드가 한꺼번에  생성됨

\- 모든 요청 사항이 반영 되었는지 확인하기 어려움

\- 이후 수정 과정에서도 불편함이 따름



##### &#x20;단계별로 나누면? → 컴퓨팅 사고의 분해



\- 각 단계마다 요청→생성→확인→수정 의 짦은 사이클 반복

\- 변경 위험을 줄이고, 구조와 품질을 차근차근 확보

\- 점점 원하는 형태에 가까운 웹페이지를 완성



##### &#x20;핵심 키워드1



\- 프롬프트(Prompt) — AI와 소통하는 유일한 창구, AI 성능을 좌우하는 핵심

\- 5W1H— Why, Who, What, When, Where, How 체계적 사고의 틀

\- PRD — 개발자들이 사용하는 제품 기획서 (Product Requirements Document)

\- 템플릿 — 효과적인 프롬프트를 위한 재사용 가능한 질문 구조



##### &#x20;핵심 키워드2



\- 4단계 전략 - 뼈대 → 기능 → 디자인 → 점검 순서로 AI에게 요청

\- 섹션(Section) - 웹페이지를 주제별로 구분한 구역 단위

\- 뼈대 - 기본 HTML 구조만으로 틀을 잡는 단계

\- 기능 - 실제 데이터를 채워 넣고 동작을 추가하는 단계

\- 디자인 - 색상, 레이아웃, 시각적 요소를 적용하는 단계

\- 점검 - 누락된 기능이나 오류를 수정하는 최종 단계
